- (direction) Paradigm = client-BaaS with a serverless AI-gateway: three thin client surfaces (Expo elder app + one role-routed React web app) over one Supabase backend (Postgres+Auth+Storage+RLS); all secret-bearing/AI/rule-enforcing logic in Edge Functions. Rx flow is a pipeline: capture -> parse -> verify(gate) -> schedule.
- (constraint) Inherited from PRD v1-core-locked: design locks DC-1..DC-8 bind; v1 = English+Malay, single spoken consent at onboarding, caregiver-led onboarding only, 5-week core scope. Deferred FRs (PRD Sec 11) out of scope for this spine.
- (decision) AD-1 [ADOPTED] Single shared datastore = source of truth. Binds: all surfaces. Prevents: per-surface divergent data models / sync bugs. Rule: elder record is the hub; every entity FKs to elder; no surface persists its own authoritative copy.
- (decision) AD-2 [ADOPTED] Secrets and AI calls only server-side. Binds: elder app, web, all AI. Prevents: Gemini key leaking in a shipped bundle; clients coupling to Gemini directly. Rule: clients call own Supabase Edge Functions; only Edge Functions hold keys and call Gemini/3rd-parties.
- (decision) AD-3 [ADOPTED] Confirmation gate enforced in data not UI. Binds: hero, reminders, safety. Prevents: an unverified Rx becoming a live reminder (DC-3, NFR2). Rule: reminder row creatable only when medication.confirmed=true, enforced by DB trigger; confirmed defaults false; only linked caregiver/elder may flip.
- (decision) AD-4 [ADOPTED] Pill appearance is human-confirmed data never model output. Binds: hero FR7. Prevents: model-guessed pill identity (DC-3). Rule: medication.appearance/appearance_photo set only via caregiver onboarding capture; the parse Edge fn must never write appearance.
- (decision) AD-5 [ADOPTED] Voice I/O on-device; only reasoning crosses network. Binds: elder app conversation, NFR5. Prevents: per-minute cloud voice cost + latency blowout. Rule: STT/TTS behind a swappable voice interface, default on-device (expo-speech / expo-speech-recognition); network payload is text (fallback: one audio->Gemini call); provider swap must not change callers.
- (decision) AD-6 [ADOPTED] Schema-validated structured output for safety-bearing AI. Binds: Rx parse, passive-log significance. Prevents: brittle free-text parsing -> silent wrong dose (NFR2). Rule: safety-bearing Gemini calls declare a responseSchema returning per-field confidence; low-confidence field is flagged, never auto-accepted.
- (decision) AD-7 [ADOPTED] Role-scoped access via Row-Level Security at the DB. Binds: all surfaces, consent FR3/FR42. Prevents: cross-elder data reads; UI-only auth bugs. Rule: every table has RLS keyed on linked elder + role; doctor = read-only and consent-gated; UI routing is not the security boundary.
- (decision) AD-8 [ADOPTED] Reminders fire from on-device local scheduling. Binds: reminders, NFR10. Prevents: missed doses under flaky/no connectivity; dependence on push infra. Rule: confirmed medication generates local scheduled notifications on the device; server is not in the firing path.
- (decision) AD-9 No clinical judgement is an architectural constraint. Binds: conversation, safety FR28/29. Prevents: model-driven triage/diagnosis (DC-1/DC-2). Rule: red-flag detection is a deterministic keyword gate BEFORE the model with a fixed response; the model is never the decider of medical safety; conversation Edge fn system prompt encodes the locks.
- (decision) AD-10 Memory retrieval is structured SQL query not vector RAG in v1. Binds: memory/retention FR19. Prevents: over-engineering a vector store; unpredictable retrieval. Rule: memory answered by typed query over memory table (type+recency+keyword); pgvector deferred.
- (decision) AD-11 Store extracted facts not raw audio. Binds: all logging, privacy NFR6. Prevents: sensitive raw-audio retention / PDPA exposure. Rule: no table stores audio; only transcripts/extractions persist; audio buffers discarded after processing.
